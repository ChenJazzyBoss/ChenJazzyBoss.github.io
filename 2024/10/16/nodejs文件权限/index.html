npm 文件权限

|浏览量:

npm 文件权限问题导致无法创建新项目

报错信息

一下报错信息是在我创建一个测试vue项目中出现的，主要是安装包时出现了没有对文件修改的权限报错。

通过以下方法解决并成功创建vue项目。

手动修改文件夹权限

可以手动修改项目目录或 npm 缓存目录的权限，使你拥有写入权限。具体步骤如下：

步骤:

  1. 打开文件资源管理器，找到要修改权限的文件夹，比如我的 E:\nodejs\node_cache、E:\nodejs\node_global。
  2. 右键单击该文件夹，选择 **”属性”**。
  3. 进入 “安全” 标签页，点击 “编辑” 按钮。
  4. 在新窗口中，找到你的用户名，确保你对该文件夹有 “完全控制” 权限。如果没有，请勾选该选项。
  5. 如果你的用户名不在列表中，点击 **”添加”**，输入你的用户名，然后授予 “完全控制” 权限。
  6. 点击 应用 和 确定，保存设置。

回顾

首先，我觉得是实验室电脑有所限制，注意到基本大多数用户都是没有完全控制的权限的。

其二，很可能是我之前自定义安装全局包和缓存的地址进行了更改，导致了这些错误。
